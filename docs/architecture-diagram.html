Key Data Flows
Critical Workflows
A. Signup → VM Provisioning
1
POST /auth/signup → create Tenant, Workspace, Bot, ProvisioningJob atomically
2
Provisioning-worker picks up job (queued → validating → creating_resources → bootstrapping_vm)
3
Azure VM provisioned, Docker installed, bot image pulled with Key Vault env vars
4
Runtime registers heartbeat → RuntimeInstance status → ready
5
Dashboard shows live provisioning progress card via SSE
B. Connector Action Execution
1
Runtime task requests action (e.g., create_comment on Jira)
2
Role-policy checked via connector-gateway · heuristic risk classification
3
LOW → execute immediately. MEDIUM/HIGH → approval-service intake queue
4
Dashboard shows pending by risk level · user approves/rejects with reason
5
Decision → connector dispatch (exponential backoff) → ConnectorAction audit event
C. Payment → Contract → VM
1
User selects plan → POST /billing/orders (Order: pending)
2
Stripe/Razorpay webhook verifies HMAC → Order.status → paid
3
pdfkit generates contract PDF · Zoho Sign OAuth: upload, submit for signature
4
Zoho Sign webhook on completion → POST /webhooks/zoho-sign
5
Order: signed → ProvisioningJob created → VM provisioning begins
D. Multi-LLM Auto Failover
1
Execution engine: heuristic risk classification (confidence threshold: 0.6)
2
If confidence < 0.6 → LLM resolution required via createLlmDecisionResolver
3
Iterate health-score-ordered provider chain (5-min rolling error rate + latency)
4
On failure → record ProviderFailoverTraceRecord[] → try next provider
5
Heuristic fallback if all fail · all traces logged for debugging
E. Browser Action Audit (Tier 11)
1
Desktop action triggered (workspace_browser_open, workspace_app_launch, etc.)
2
DESKTOP_OPERATOR=mock → short-circuit; native → Playwright automation
3
action-observability.executeObservedAction() intercepts · screenshot before
4
Action executed → screenshot after · DOM hash · network log · correctness assertion
5
All data → BrowserActionEvent in AgentSession · retention policy applied
F. Skills Crystallization (Hermes)
1
Task completes successfully
2
SkillsRegistry.crystallize() extracts template from execution
3
State: draft → active lifecycle
4
Future tasks call findMatching() to accelerate similar requests
5
Skill composition engine chains multi-skill dependency DAGs
Security Boundaries & Critical Invariants
Security Model — 11 Critical Boundaries
🔑
No Secrets in DB
Only Azure Key Vault references persisted — no raw connector secrets
🏠
Workspace RLS
All routes scoped by workspace at session level (Row-Level Security)
🔒
Approval Immutability
409 on re-decision; createdAt/decidedAt immutable after creation
☠️
Kill-Switch
30-second control window; incident-reference-gated resume
🎯
CSRF Nonce
OAuth flows validate state nonces before token exchange
⏱️
Timing-Safe Comparison
Password hashing prevents user enumeration attacks
🔄
Exponential Backoff
Transient connector failures retry safely without amplification
📁
safeChildPath Sandbox
All file/shell operations sandboxed to workspace root
🔐
Plugin Trust Verification
Cryptographic signature check before allowlist for external plugins
📜
mTLS Certificate Verification
Federation security in connector-gateway for agent-to-agent trust
🫥
PII-Strip Middleware
Data redaction before any external dispatch or connector call
📊
Audit Append-Only
AuditEvent model has no update/delete operations in schema
